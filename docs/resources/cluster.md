# velodb_cluster (Resource)

Use the *velodb_cluster* resource to create and manage clusters within a VeloDB Cloud warehouse.

A cluster is a compute unit within a warehouse. Each warehouse can contain multiple clusters of different types (`SQL`, `COMPUTE`, `OBSERVER`). Clusters can be independently scaled, paused, and resumed.

Key capabilities:

- **Declarative lifecycle** — set `desired_state` to `running` or `paused` instead of imperative start/stop commands
- **Auto-pause** — configure idle timeout for automatic pause to save costs
- **Resize** — change `compute_vcpu` and `cache_gb` to scale in-place
- **Billing control** — switch between `on_demand` and `monthly` billing
- **Connection info** — public/private endpoints and ports exposed as computed attributes

## Example Usage

### Basic Compute Cluster

```terraform
resource "velodb_cluster" "etl" {
  warehouse_id   = velodb_warehouse.main.id
  name           = "compute-etl"
  cluster_type   = "COMPUTE"
  zone           = "cn-beijing-k"
  compute_vcpu   = 4
  cache_gb       = 100
  billing_method = "on_demand"
  desired_state  = "running"

  auto_pause {
    enabled              = true
    idle_timeout_minutes = 15
  }

  timeouts {
    create = "20m"
    update = "20m"
  }
}

output "etl_endpoint" {
  value = velodb_cluster.etl.connection_info[0].public_endpoint
}
```

### Cost-Optimized Dev Cluster (Paused by Default)

```terraform
resource "velodb_cluster" "dev" {
  warehouse_id   = velodb_warehouse.main.id
  name           = "compute-dev"
  cluster_type   = "COMPUTE"
  zone           = "cn-beijing-k"
  compute_vcpu   = 4
  cache_gb       = 100
  billing_method = "on_demand"
  desired_state  = "paused"

  auto_pause {
    enabled              = true
    idle_timeout_minutes = 5
  }
}
```

### Prepaid Monthly Cluster

```terraform
resource "velodb_cluster" "prepaid" {
  warehouse_id       = velodb_warehouse.main.id
  name               = "sql-primary"
  cluster_type       = "SQL"
  zone               = "cn-beijing-k"
  compute_vcpu       = 16
  cache_gb           = 800
  billing_method     = "monthly"
  period             = 1
  period_unit        = "Month"
  auto_renew_enabled = 1
  desired_state      = "running"

  auto_pause {
    enabled = false
  }
}
```

### Resize a Cluster

To resize, change `compute_vcpu` and/or `cache_gb`. The provider patches the cluster and waits for the resize to complete:

```terraform
resource "velodb_cluster" "etl" {
  # ...
  compute_vcpu = 8    # was 4
  cache_gb     = 500  # was 100
  # ...
}
```

```
$ terraform plan
~ velodb_cluster.etl
    compute_vcpu: 4 → 8
    cache_gb:     100 → 500
```

### Pause and Resume

To pause a running cluster, change `desired_state`. The provider calls the cluster actions API and waits for the target state:

```terraform
resource "velodb_cluster" "etl" {
  # ...
  desired_state = "paused"  # was "running" — triggers POST /actions {"action":"pause"}
  # ...
}
```

To resume:

```terraform
resource "velodb_cluster" "etl" {
  # ...
  desired_state = "running"  # was "paused" — triggers POST /actions {"action":"resume"}
  # ...
}
```

### Multiple Clusters in One Warehouse

```terraform
resource "velodb_warehouse" "main" {
  name            = "analytics"
  deployment_mode = "SAAS"
  cloud_provider  = "aliyun"
  region          = "cn-beijing"

  admin_password         = var.admin_password
  admin_password_version = 1

  initial_cluster {
    name         = "sql-primary"
    compute_vcpu = 16
    cache_gb     = 800
  }
}

resource "velodb_cluster" "etl" {
  warehouse_id   = velodb_warehouse.main.id
  name           = "compute-etl"
  cluster_type   = "COMPUTE"
  compute_vcpu   = 32
  cache_gb       = 1600
  desired_state  = "running"

  auto_pause {
    enabled              = true
    idle_timeout_minutes = 15
  }
}

resource "velodb_cluster" "dev" {
  warehouse_id   = velodb_warehouse.main.id
  name           = "compute-dev"
  cluster_type   = "COMPUTE"
  compute_vcpu   = 4
  cache_gb       = 100
  desired_state  = "paused"

  auto_pause {
    enabled              = true
    idle_timeout_minutes = 5
  }
}

# Connection info for the ETL cluster
output "etl_jdbc_endpoint" {
  value = "${velodb_cluster.etl.connection_info[0].public_endpoint}:${velodb_cluster.etl.connection_info[0].listener_port}"
}
```

## Schema

### Required

- `cache_gb` (Number) Cache capacity in GB.
- `cluster_type` (String) Cluster type: `SQL`, `COMPUTE`, or `OBSERVER`. Changing this forces a new resource.
- `compute_vcpu` (Number) Compute capacity in vCPUs. Changing this triggers a resize operation.
- `name` (String) Cluster display name.
- `warehouse_id` (String) Parent warehouse identifier. Changing this forces a new resource.

### Optional

- `auto_pause` (Block List, Max: 1) Auto-pause configuration. When enabled, the cluster automatically pauses after the idle timeout. (see [below for nested schema](#nestedblock--auto_pause))
- `auto_renew_enabled` (Number) Auto-renew flag for prepaid billing.
- `billing_method` (String) Billing method: `on_demand` or `monthly`.
- `desired_state` (String) Desired cluster lifecycle state. Setting this triggers the corresponding cluster action via the API. Valid values: `running` (triggers `resume` if paused), `paused` (triggers `pause`). Defaults to `running` when computed from API status.
- `period` (Number) Prepaid subscription length. Used with `period_unit`.
- `period_unit` (String) Period unit: `Month`, `Year`, or `Week`.
- `timeouts` (Block, Optional) (see [below for nested schema](#nestedblock--timeouts))
- `zone` (String) Availability zone. Changing this forces a new resource. Computed from the API if not set.

### Read-Only

- `cloud_provider` (String) Cloud provider inherited from the parent warehouse.
- `connection_info` (Attributes List) Cluster connection endpoints. (see [below for nested schema](#nestedatt--connection_info))
- `created_at` (String) Cluster creation time in ISO 8601 / RFC 3339 format.
- `disk_sum_size` (Number) Current disk capacity in GB.
- `expire_time` (String) Cluster expiration time when available.
- `id` (String) Cluster identifier (e.g., `c-1997tallv8chbkdhej`).
- `pay_type` (String) Billing type: `PostPaid` or `PrePaid`.
- `region` (String) Cloud region inherited from the parent warehouse.
- `started_at` (String) Cluster start time in ISO 8601 / RFC 3339 format.
- `status` (String) Current observed cluster status. One of: `Creating`, `Running`, `Resizing`, `Adjusting`, `Upgrading`, `Suspending`, `Resuming`, `Stopping`, `Starting`, `Restarting`, `Deleting`, `Suspended`, `Stopped`, `Deleted`, `CreateFailed`.

## Import

Import is supported using the following syntax:

```shell
# Clusters can be imported by specifying warehouse_id/cluster_id.
terraform import velodb_cluster.example ALBJRXRG/c-m2w789x8kghgpapgaz
```

Or using the Terraform 1.5+ import block:

```terraform
import {
  to = velodb_cluster.example
  id = "ALBJRXRG/c-m2w789x8kghgpapgaz"
}
```

~> **Note:** The `compute_vcpu`, `cache_gb`, `billing_method`, `desired_state`, and `auto_pause` attributes cannot be read from the API and will not be populated after import. You must add them to your configuration manually.
